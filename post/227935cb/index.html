    前言

    这里说的CSF文件指的是命令与征服系列用来存放字符串资源的文件
    这里的代码默认情况下都是C#语言的
    其实 我一共写过3个CSF编辑器 后来…都在还没开发完就弃坑了!

    C#的Char是占两字节的
    unsafe指这段代码不受GC管理

    文件结构

    不安全警告
    文件结构参考Modenc[1] 感谢miRoox的翻译[2]
    CSF文件的结构其实很简单

        /// <summary>
    /// 表示CSF文件的结构
    /// </summary>
    public unsafe struct CsfFile
    {
        public CsfHead Head;// 文件头部信息
        public CsfLabel* Body;// 文件主体内容
    }

    文件头

        /// <summary>
    /// 表示CSF文件头的结构
    /// </summary>
    public unsafe struct CsfHead
    {
        public fixed char Flag[4];// 是ASCII字符串 始终为0x20 46 53 43 即为" FSC" 确定这是一个CSF文件 否则游戏不会读取这个文件
        public int Version;// 文件版本 通常为3
        public int LabelCount;// 标签总数 这个CSF文件含有的标签总数
        public int ValueCount;// 字符串总数 同上 CSF文件可以一个标签后面接多个字符串(不过没什么用)
        public fixed byte Unknown[4];// 用途不明的4个字节
        public int Language;// 文件的语言信息
    }

    文件语言

    标签

        /// <summary>
    /// 表示CSF标签的结构
    /// </summary>
    public unsafe struct CsfLabel
    {
        public fixed byte Flag[4];// 是ASCII字符串 始终是" LBL" 表示这是一个标签
        public int Count;// 这个标签一共包括多少个字符串值
        public int NameLength;// 这个标签的名字长度
        public byte* Name;// 是ASCII字符串 这个标签的名字
        public CsfValue* Values;// CSF字符串结构
    }

    字符串结构

        /// <summary>
    /// 表示CSF字符串的结构
    /// </summary>
    public unsafe struct CsfValue
    {
        public fixed byte Flag[4];// 是ASCII字符串 共有两种值" RTS"和"WRTS"
        public int Length;// 表示字符串数量 因为Unicode是两字节长度所以要左移一位使用
        public char* Content;// 是经过按位取反的Unicode字符串
        public int? ExtraLength; // 额外内容长度 仅在Flag为"WRTS"时可用
        public byte* ExtraContent;// 是ASCII字符串 额外内容
    }

    还原字符串

        public unsafe static byte[] Decoding(int ValueLength, void* ValueData)
    {
        int ValueDataLength = ValueLength << 1;
        byte[] data = new byte[ValueDataLength];
        Marshal.Copy(new IntPtr(ValueData), data, 0, ValueDataLength);
        return Decoding(data);
    }
    public static byte[] Decoding(byte[] ValueData) => ValueData.Select(i => (byte)~i).ToArray();

    写在最后

    如果你是直接复制粘贴的我上面的代码的话 那你可以直接删掉了 这个写出来用会有严重的性能问题的
    结构可是值类型 赋值的时候是传值不是传引用
    这里使用结构是为了更直观的展示文件结构
